# Spring Security

### Authentication(인증)
- 애플리케이션에 접근하려는 주체(Principal)의 신원을 확인한느 과정
- 일반적으ㄹ 사용자가 제공한 자격 증명(Credentials)이 시스템에 저장된 정보와 일치하는지 검증하는 작업을 의미
### Authorization(인가)
- 인증된 주체에게 부여된 권한(Granted Authorities)을 기반으로, 특정 리소스(Resource) 또는 동작에 대한 접근 여부를 결정하는 과정
- 인증이 선생되어야 하며, 웹 요청 URL 단위의 접근 제어뿐만 아니라 메서드 단위(@PreAuthorize)의 세밀한 접근 제어도 포함

### Architecture: Filter Chain
- `Spring Security`의 `Servlet-based` 지원은 기본적으로 `Servlet Filters`를 기반으로 동작 

#### DelegatingFilterProxy 와 FilterChainProxy
- 서블릿 컨테이너의 필터 생명주기와 Spring의 `ApplicationContext`를 연결하는  브릿지 역할을 수행
- 사용자 HTTP 요청(`HttpServletRequest`)이 `DispatcherServlet`에 도달하기 전에 프록시를 거침

#### SecurityFilterChain
- `FilterChainProxy`에 의해 관리되는 여러 개의 `SecurityFilter` 인스턴스들의 집합
- 각 필터는 단일 책임 원칙에 따라 고유한 작업
  - CORS 처리
  - CSRF 방어
  - 세션 관리
  - 폼 로그인
- `Exception Translation`
  - 필터 체인을 거치는 동안 권한 부족이나 인증 실패가 발생하면, `ExceptionTranslationFilter`가 이를 포착하여 `AuthenticationException` 또는 `AccessDeniedException` 형태의 HTTP 응답으로 변환하여 클라이언트에게 반환

### 인증 모델: SecurityContextHolder 아키텍처
*Sping Security*가 인증된 사용자의 세부 정보를 저장하고 참조하는 계층적 데이터 모델

#### `Authentication`(인증 객체)
- 현재 접근하는 주체의 인증 상태를 나타내는 인터페이스
- **Principal(주체)**: 주로 식별자
- **Credentials**(자격 증명): 비밀번호(보통 인증 완료 후 유출 방지를 위해 삭제) 
- **Authorities**(권한): 주체에게 부여된 역할 및 권한의 컬렉션(`GrantedAuthority`)

#### `SecurityContextHolder`(컨텍스트 보관소)
- Spring Security 인증 모덴의 심장부
- 기본적으로 `ThreadLocal` 전략을 사용하여, 현재 실행 중인 스레드 내의 어디서든 동일한 `SecurityContext`에 접근할 수 있도록 상태를 저장

# `Session` 기반과 `JWT`기반 아키텍처의 차이
상태 관리 전략(State Management Strategy)의 관정에서 두 방식을 대조

### Session 기반 인증 (Stateful Architecture)
- 인증이 성공하면 서버는 `SecurityContext`를 `HttpSessionSecurityContextRepository`를 통해 서버의 메모리(세션)에 Persist
- 서버는 클라이언트에게 세션 식별자(`JSESSIONID`)를 쿠키로 발급하여, 후속 요청 시 필터 체인의 `SecurityContextPersistenceFilter`가 쿠키를 읽어 세션에서 컨텍스트를 복원

### JWT 방식 인증 (Stateless Architecture)
- 무상태성 적용: `SeesionCreationPolicy.STATELESS`로 설정, 서버가 `SecurityContext`를 HTTP 세션에 저장하거나 캐시하지 않도록 강제
- 커스텀 필터 주입(`addFilterBefore`): 기존의 폼 로그인 필터(`UsernamePasswordAuthenticationFilter`)의 역할을 무효화하고, 매 요청마다 HTTP 헤더(`Authorization: Bearer <token>`)에서 토큰을 추출 및 검증하는 커스텀 필터(`OncePerRequestFilter` 구현체)를 필터 체인 앞단에 배치
- 동적 컨텍스트 할당: 커스텀 필터에서 토큰의 서명과 만료 여부를 검증(`Verify`)한 후, 유효한 토큰에서 추출한 Claim(사용자 정보)을 바탕으로 `Authentication`객체를 생성하여 해당 요청(Request)을 처리하는 스레드의 `SecurityContextHolder`에 일회성으로 할당, 요청 처리가 끝나면 상태 유지되지 않는다.
